Préambule

 

Ce document a pour objet de permettre la réalisation des exercices et des TP du master tried sur un projet de plateforme collaborative Jupyter en langage python. Nous avons retenu python 3 pour la réalisation de ces travaux. Ceux qui préfèreraient python 2 restent libres de ce choix, à charge pour eux de coder toutes les méthodes dont ils pourraient avoir besoin. De même les étudiants qui le souhaitent, peuvent modifier à leur convenance les fonctions qui sont mises à leur disposition, ou créer leurs propres fonctions. Incidemment, Il est encore possible d’utiliser les mêmes algorithmes ou d’autres « dénichables » dans d’autres packages python.

 

Le propos de ce document est de montrer, aux travers de différents exemples, l’usage des méthodes que nous avons développées, et qui sont listées ci-après. Il n’était pas question ici, de présenter tous les exercices dans le moindre détail, mais au contraire de vérifier, dans les grandes lignes, que les principales fonctions utilisées pouvaient être adaptées pour continuer à rendre un service minimal.

  shownpat2D   : Affichage d'un vecteur sous forme d'une image 2D

  gen2dstd     : Génération de données simulées 2D selon une loi N(m,s)

  intarray2str : Transformation d'un tableau d'entiers en tableau de strings

  vdist        : Calcul de distance

  unique       : Retourne les occurrences, sans répétition, d'un tableau 2D

  ismember     : Indique si un vecteur (ligne) est contenu dans un tableau 2D

  unifpdf      : Fonction de densité de la loi uniforme

cloud        : Nuage utilisable pour les individus comme pour les variables

corcer       : Cercle des corrélation

Avant propos : C’est le module sompy que nous avons utilisé pour les cartes de Kohonen.  Ce module a été écrit pour python 2. Il est disponible là-> https://github.com/sevamoo/SOMPY. Nous vous recommandons de consulter cette adresse car l’auteur (Vahid Moosavi) apporte des précisions importantes sur les restrictions de son code (par exemple, les connexions hexagonales ne sont actuellement pas gérées). Il indique également qu’il n’y a pas de documentation, et que c’est le code lui-même qui en fait office. On trouve également des liens qui montrent l’utilisation de son code, nous les reportons ci-dessous :

http://bit.ly/1eZvaCM

http://bit.ly/1DHdLpn

http://bit.ly/1zfn77s

http://bit.ly/1ujaD36

et un lien où l’auteur présente son travail :

http://vahidmoosavi.com/2014/02/18/a-self-organizing-map-som-package-in-python-sompy/

 

Etant donné que nous avons choisi python 3, nous avons traduit sompy.py en python 3 (avec le code de traduction 2to3.py). Nous n’y avons apporté par ailleurs, essentiellement les adaptations suivantes  de la méthode train :

- courcircuitage de l’exécution en mode parallèle

- passage de paramètres pour les 2 phases d’apprentissage : nombre d’itérations, températures initiale et finale.

 

Nous avons rebaptisé notre adaptation de sompy.py en triedsompy.py. A ce module, nous avons adjoint un module, triedctk.py, qui contient nos propres méthodes complémentaires. Il faut donc un import de ces deux modules pour obtenir les résultats présentés ci-après qui montrent plus particulièrement le rendu de nos propres méthodes de visualisation.
